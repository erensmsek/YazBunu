# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

A graduation project (BTK Bitirme Projesi): a fully local speech-to-text application. No cloud hosting, no Hugging Face Spaces, no ZeroGPU â€” everything runs on the user's MacBook Air M4 (16GB). Hugging Face Hub is used only as a model registry (downloading weights once, then running entirely offline/locally).

Stack:
- **Transcription**: `large-v3-turbo` Whisper, 8-bit quantized, via `mlx-whisper` (Apple Silicon-native, Metal-accelerated) â€” chosen over `whisper.cpp` and over PyTorch+MPS.
- **Translation**: `facebook/nllb-200-distilled-600M` via `transformers`, arbitrary sourceâ†’target language (not just Xâ†’English like Whisper's built-in translate task).
- **Speaker diarization**: `pyannote/speaker-diarization-3.1` via `pyannote.audio` (gated model, currently blocked â€” see Current Status).
- **Summarize + polish (LLM)**: `Qwen/Qwen2.5-7B-Instruct` (BF16) via **Hugging Face Inference API** (`llm.py`). This is the one **deliberate exception** to the "everything local" rule â€” see gotcha #7. The local 4-bit MLX version was tested and rejected (too slow ~35s, hallucinations, degraded Turkish).
- **Backend**: FastAPI (`server.py`), single process, no build step.
- **Frontend**: hand-written HTML/CSS/JS in `static/` â€” deliberately **not** Gradio/Streamlit. Chosen for a more polished, custom-designed UI since the app never needs to be deployed to Spaces.

## Architecture

`server.py` is the single entry point (FastAPI). It auto-detects `IS_APPLE_SILICON` at import and picks a provider per request via `_use_local(mode)` (see gotcha #8 for the dual-mode design). Routes:
- `GET /` â€” serves `static/index.html`.
- `GET /config` â€” `{apple_silicon, default_mode}`. The frontend uses this to decide whether to show the Lokal/API mode toggle (Apple Silicon only).
- `POST /transcribe` â€” multipart form: `file` (audio), optional `target_lang` (ISO 639-1), optional `diarize` (bool), optional `mode` ("api"|"local"), optional `api_key` (Groq). Pipeline: transcribe (local `mlx_whisper` if `_use_local`, else `groq_api.transcribe`) â†’ build segment list â†’ if `diarize`, merge speaker labels from `diarize.py` (always local) â†’ if `target_lang` differs from detected language, translate (local `translate.py` NLLB if `_use_local`, else `groq_api.translate_texts`). Returns `{language, text, segments[], translation?}`. Groq errors surface as HTTP 502 with a user-facing Turkish message.
- `POST /summarize` â€” JSON body `{text, api_key?}` â†’ `{summary}`. `_run_llm` routes to `groq_api.summarize` when `api_key` is present, else falls back to `llm.summarize()` (HF Inference API). See gotchas #7 and #8.
- `POST /polish` â€” JSON body `{text, api_key?}` â†’ `{polished}` (Markdown). Same routing via `_run_llm`. The frontend renders the Markdown client-side (minimal renderer in `script.js`) and lets it be exported via `/export`.
- `POST /export` â€” JSON body `{format, text, segments[]}` â†’ downloadable file via `export.py`'s `EXPORTERS` dict. Called twice by the frontend (original + translated).
- `@app.on_event("startup")` â€” `ensure_model_weights_alias()`; now **guarded by `IS_APPLE_SILICON`** so the 824MB MLX model isn't downloaded on API-only platforms. See gotcha #1.

Supporting modules (each is a standalone, lazily-initialized wrapper â€” same pattern: a module-level `_model`/`_pipeline` singleton populated on first use):
- `export.py` â€” pure formatting functions, no I/O: `to_txt`, `to_srt`, `to_vtt`, `to_md`, `to_json`, all registered in `EXPORTERS: {format: (fn, media_type)}`.
- `translate.py` â€” NLLB wrapper. `LANG_MAP` (ISO 639-1 â†’ FLORES-200 code) is the single source of truth for which languages the UI's translate dropdown should offer â€” extend both `LANG_MAP` here and the `<select>` in `static/index.html` together when adding a language. `translate_texts()` batches all segment texts into one `model.generate()` call rather than one call per segment. Note: `LANG_MAP` currently has 20 entries, but the `<select id="targetLang">` in `index.html` only exposes 12 of them (missing `nl`, `pl`, `sv`, `uk`, `el`, `hi`, `fa`, `ro`) â€” relevant when picking up New Features item 8 (language picker UI).
- `diarize.py` â€” pyannote wrapper. `diarize()` returns raw speaker turns; `assign_speakers()` merges them onto Whisper segments by max-time-overlap and relabels pyannote's raw IDs (`SPEAKER_00`, ...) into sequential `KonuĹźmacÄ±_1`, `KonuĹźmacÄ±_2`, ... in order of first appearance (deliberately no gender/identity detection â€” sequential labels only, per project requirements).
- `groq_api.py` â€” **NOT local**: Groq (OpenAI-compatible) wrapper over plain `requests`, the single provider for **API mode**. `transcribe()` (Whisper via `/audio/transcriptions`, `verbose_json` â†’ same dict shape as `mlx_whisper`), `translate_texts()` (batch translate through a chat LLM with a strict JSON-in/JSON-out prompt to preserve segment alignment â€” the API-mode replacement for NLLB), `summarize()`/`polish()` (same prompts/contract as `llm.py`). One Groq key covers all of these. Raises `GroqError` (user-facing Turkish messages) on 401/403/429/other. Model ids are module constants (`TRANSCRIBE_MODEL`, `LLM_MODEL`) â€” confirm against Groq's live catalog. See gotcha #8.
- `llm.py` â€” **NOT local** (unlike every other module): thin wrapper over `huggingface_hub.InferenceClient` calling `Qwen/Qwen2.5-7B-Instruct` via the `featherless-ai` provider (BF16, full precision). Same lazy-singleton pattern (`_client`). `summarize()` â†’ short Turkish summary (narrow `max_tokens` + explicit "sadece TĂĽrkĂ§e" to prevent the observed Chinese-drift glitch); `polish()` â†’ Markdown-formatted rewrite with title. Both use a system+user chat template and `temperature=0.4`. Needs `HF_TOKEN` in `.env`. See gotcha #7 for why this is API-based, not local.

`static/` â€” no framework, no build step:
- `index.html` â€” settings row (target-language `<select>`, diarize `<checkbox>`) shared by both input modes, a tab switcher (Mikrofon / Dosya YĂĽkle), and a results section with two parallel blocks (original transcript, optional translation), each with its own segment list and export button row.
- `script.js` â€” `MediaRecorder`-based mic capture (see gotcha #4), drag-and-drop + `<input type=file>` upload, renders segments (with speaker-tag prefix when present) and drives `/export` downloads via `Blob` + a synthetic anchor click. Also: live mic **waveform** via Web Audio `AnalyserNode` drawn on a `<canvas>` (`startWaveform`/`stopWaveform`, `requestAnimationFrame` bar chart); `Ă–zetle`/`BaĹźlÄ±klandÄ±r & Ä°yileĹźtir` buttons that POST the transcript to `/summarize` & `/polish`; and a **minimal Markdown renderer** (`renderMarkdown()` â€” handles `#`/`##`/`###`, `- `/`1.` lists, `**bold**`, `*italic*`, `` `code` ``) for the LLM output. No external Markdown lib (keeps the no-build-step constraint).
- `style.css` â€” CSS custom properties, `prefers-color-scheme` for dark/light, no framework. Refreshed to a modern rounded light theme (indigo accent, gradient-bordered AI cards, pill settings, animated record button + waveform). Light is the primary/designed-for theme; dark is a maintained fallback.

## Key technical decisions & gotchas

1. **Model weights filename mismatch (already fixed â€” don't reintroduce)**: `mlx-community/whisper-large-v3-turbo-8bit` ships weights as `model.safetensors`, but `mlx_whisper`'s `load_model()` only looks for `weights.safetensors` / `weights.npz`. `_ensure_mlx_weights()` in `server.py` hardlinks one name to the other (no data duplication, no re-download). **Since 2026-07-16 this runs lazily**, not on app startup â€” it's called from inside `_transcribe_local()`, guarded by a `_mlx_weights_ready` flag, so the 824MB download only happens the first time a request actually uses Lokal mode (see gotcha #8; this replaces the old startup-hook version that downloaded unconditionally on every boot on Apple Silicon, even for API-mode-only users â€” a real bug caught and fixed the same day it was introduced). If `MODEL_REPO` is ever pointed at a different `mlx-community` quantization, check its actual filenames first via `huggingface.co/api/models/<repo>` â€” some repos (e.g. `whisper-large-v3-turbo-q4`, the plain `whisper-large-v3-turbo`) already use `weights.safetensors`/`weights.npz` natively and need no fix; others (`-8bit`, `-4bit`, `-asr-*` variants) use `model.safetensors` and need this same alias trick.

2. **Hugging Face's "Xet" CDN (`cas-bridge.xethub.hf.co`) has been unreliable in this environment**: large public-model downloads have repeatedly stalled for many minutes at a fixed byte offset before either completing or requiring a retry. `hf_xet` (the accelerated Xet client) has been uninstalled to force the classic HTTP download path â€” keep it uninstalled (don't let `pip install -r requirements.txt` silently pull it back in via `transformers`/`huggingface_hub` extras). Also always export `HF_HUB_DISABLE_XET=1` as a second guard. If a download appears stuck, check whether `~/.cache/huggingface/hub/**/*.incomplete` is actually growing over ~30s before concluding it's broken.

3. **Gated diarization model â€” currently blocked, see Current Status.** `pyannote/speaker-diarization-3.1` requires accepting gated terms for *both* itself and its dependency `pyannote/segmentation-3.0` (two separate gates) at huggingface.co while logged in, plus an `HF_TOKEN` (read scope) in `.env`. Acceptance status is visible at `https://huggingface.co/settings/gated-repos`. Also note: `pyannote.audio`'s `Pipeline.from_pretrained()` takes a `token=` kwarg, not `use_auth_token=` (older API, removed in the installed version).

4. **Mic recording quality**: `script.js`'s `getUserMedia` call explicitly sets `echoCancellation`, `noiseSuppression`, and `autoGainControl` to `false`, and `MediaRecorder` is given `audioBitsPerSecond: 128000`. Browser defaults are tuned for VoIP calls and were measurably degrading Whisper's transcription accuracy (especially non-English) on real microphone input versus a clean uploaded file â€” don't revert this without re-testing accuracy.

5. **`mlx_whisper.transcribe()` caches its loaded model** as a class attribute (`ModelHolder`, keyed by the `path_or_hf_repo` string) across calls within the same process â€” as long as `MODEL_REPO` doesn't change between requests, the model loads once and stays warm. No additional caching layer needed in `server.py`.

6. **Everything in `/transcribe` runs synchronously inside an `async def` handler** (`mlx_whisper.transcribe`, `translate_texts`, `diarize` are all blocking calls) â€” the server can't service a second request while one transcription is in flight. Acceptable for the intended single-user local use; would need `run_in_executor`/threading if concurrent usage is ever required. Note `/summarize` and `/polish` are also blocking (synchronous HTTP call to HF) â€” same single-user assumption.

7. **Summarize/polish LLM is API-based, NOT local â€” a deliberate, user-approved exception to the "everything local" principle.** We benchmarked local MLX quantizations (`mlx-community/Qwen2-1.5B`, `Qwen2.5-3B`, `Qwen2.5-7B-Instruct-4bit`) against the API. The 1.5B/3B produced broken Turkish (invented words, wrong grammar, ignored instructions); the local 7B-4bit was usable but had hallucinations (invented "HearSay" label), broken numbering, minor grammar slips, and was **slow (~35s for a polish)**. The BF16 model over HF Inference API was clearly the cleanest and fast (~7-12s), so `llm.py` calls it remotely. Provider notes: `Qwen/Qwen2.5-7B-Instruct` is served by two providers for this token â€” `together` (FP8 "-Turbo", quantized) and `featherless-ai` (original BF16). We pinned `featherless-ai` for best polish quality. The MLX 4-bit repo canNOT be served via the API (Apple-Silicon-only format) â€” only local. The user's rationale for accepting cloud here: the project's grading is partly about HF-platform usage, and for personal use the only real blockers are cost/hardware, not the local-privacy purity that governs the other three models. If privacy ever matters again, the local 7B-4bit path already works (`test_llm.py`) â€” just slower/rougher. `test_llm_hf_api.py` is a companion scratch script for probing the HF Inference API directly (model/provider as CLI args, no server needed) â€” useful when re-tuning prompts for New Features item 3; `test_output_polish.md` is a saved sample `/polish` output from an earlier run, kept for before/after comparison.

8. **Dual-mode architecture (Lokal / API) with a Settings screen â€” IMPLEMENTED 2026-07-16.** Goal: anyone cloning the repo from GitHub, on any OS, can try the app in under a minute without downloading multi-GB weights or owning Apple Silicon, while the existing fully-local Mac path is kept as an option. **What shipped**: `groq_api.py` (Groq wrapper: `transcribe`/`translate_texts`/`summarize`/`polish`, all via one Groq key, plain `requests`); `server.py` refactored (Apple-Silicon auto-detect at load, `GET /config`, `mode`+`api_key` form fields on `/transcribe`, `api_key` in `/summarize`+`/polish`, mlx import made lazy + MLX weight download skipped on non-Apple so the app boots cross-platform); Settings modal + first-launch Groq banner in `static/` (mode toggle shown only on Apple Silicon, Groq key stored in `localStorage`, "Groq anahtarÄ± al" links to `https://console.groq.com/keys`); `requirements.txt` gained a `platform_system=="Darwin" and platform_machine=="arm64"` marker on `mlx-whisper` so `pip install` no longer fails off-Apple; `.env.example` documents `GROQ_API_KEY` (UI is primary; env is a server-side fallback that `groq_api._require_key` also reads). **Routing model** (see `server.py`): transcription & translation follow the mode (local â†’ `mlx-whisper`/NLLB, api â†’ Groq); summarize & polish always go to Groq when a key is present, else fall back to the HF `llm.py` path (dev/back-compat); diarization is always local. **Bug found & fixed same day (2026-07-16)**: the initial implementation kept the old `@app.on_event("startup")` hook, which on Apple Silicon downloaded the 824MB mlx-whisper model **unconditionally on every boot**, regardless of whether the user ever picks Lokal mode â€” defeating the point of API mode being the lightweight default. Fixed by making the download lazy: `_ensure_mlx_weights()` is now called only from inside `_transcribe_local()`, gated by a `_mlx_weights_ready` flag, so it only fires the first time a Lokal-mode transcription actually runs. See gotcha #1.

**Also verified live against Groq with a real key** (2026-07-16, curl reading `.env`'s `GROQ_API_KEY` via `export $(grep ...)` â€” the key value was never printed/echoed/logged in the process). Two real bugs were caught and fixed during this pass â€” both live only in `groq_api.py`, nothing else needed to change:
   - **Language-code mismatch**: Groq's Whisper endpoint returns the detected language as a full name (`"Turkish"`), while `mlx_whisper` returns an ISO 639-1 code (`"tr"`). `server.py`'s `target_lang != language` check and the frontend's `LANG_NAMES[data.language]` lookup both expect ISO codes â€” with the raw Groq response, translation would silently misfire (always/never trigger) and the language pill would show the raw English name. Fixed with `groq_api._normalize_language()` (builds a nameâ†’code map from the existing `LANG_NAMES` dict, applied to `transcribe()`'s return value).
   - **`qwen/qwen3-32b` leaks its chain-of-thought into the response**: without an explicit setting, Groq returns the model's `<think>...</think>` reasoning inline in `message.content`, ahead of the actual answer â€” confirmed via a raw `/summarize` call whose output opened with a full English reasoning trace before the Turkish summary. Fixed by adding `"reasoning_effort": "none"` to every `_chat()` call in `groq_api.py` (Groq-documented parameter for Qwen3 models â€” disables reasoning entirely rather than just hiding it, which is also cheaper/faster for our use case since none of summarize/polish/translate need multi-step reasoning). Re-verified clean after the fix.

Both `TRANSCRIBE_MODEL="whisper-large-v3"` and `LLM_MODEL="qwen/qwen3-32b"` are confirmed live and working on Groq as of 2026-07-16. Original design decisions, still accurate:
